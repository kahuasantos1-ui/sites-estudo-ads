📦 Arrays (vetores) Java

Um array guarda vários valores do mesmo tipo em posições contíguas. O tamanho é fixo (definido na criação).

  • O índice começa em 0: o primeiro é v[0], o último é v[v.length - 1].
  • v.length dá a quantidade de posições (é atributo, sem parênteses).
  • Acessar índice inválido lança ArrayIndexOutOfBoundsException.
int[] nums = new int[5];      // 5 posições: índices 0..4
nums[0] = 10;                 // grava na 1ª posição
int[] v = {3, 7, 1, 9};       // já inicializado (length = 4)

for (int i = 0; i < v.length; i++) {
    System.out.println(v[i]); // percorre por índice
}
⚠️ Pegadinha: em array é .length (atributo). Em String é .length() (método, com parênteses).
🏷️ Enum Java

Um enum (enumeração) define um conjunto fixo de constantes nomeadas. Usa-se quando os valores possíveis são limitados e conhecidos (dias da semana, status, naipes...).

enum DiaSemana { SEG, TER, QUA, QUI, SEX, SAB, DOM }

DiaSemana hoje = DiaSemana.SEX;

switch (hoje) {
    case SAB, DOM -> System.out.println("Fim de semana!");
    default       -> System.out.println("Dia útil");
}

for (DiaSemana d : DiaSemana.values()) {   // values() = todos
    System.out.println(d);
}
💡 Por que usar: é mais seguro que int ou String soltos — o compilador só aceita os valores definidos. values() devolve todas as constantes; ordinal() dá a posição (0, 1, 2...).
🔗 Ponteiros → Referências em Java conceito-chave
📌 O mais importante: Java NÃO tem ponteiros como C/C++ (sem *, sem aritmética de endereço). Java usa referências — gerenciadas e seguras.

Valor vs. Referência

  • Tipos primitivos (int, double, char, boolean) guardam o valor direto.
  • Objetos e arrays guardam uma referência (o "endereço") ao objeto na memória (heap).
  • null = referência que não aponta pra nada.
int a = 5;          // 'a' guarda o valor 5
int[] v = {1, 2};   // 'v' guarda uma REFERÊNCIA ao array

String s = null;    // não aponta pra nada
System.out.println(s.length()); // 💥 NullPointerException
⚠️ NullPointerException: acontece quando você usa um método/atributo de uma referência que é null. É o erro mais comum em Java.

Onde isso aparece em Estrutura de Dados

Nas estruturas encadeadas (listas, pilhas e filas feitas com nós), cada nó guarda um dado e uma referência ao próximo nó — é o "ponteiro" do Java na prática.

class No {
    int valor;
    No proximo;   // referência ao próximo nó (null = fim da lista)
}
📋 Listas Java

Uma Lista é uma coleção ordenada e de tamanho dinâmico (cresce e diminui sozinha). As duas implementações principais:

TipoPor dentroBoa para
ArrayListArray redimensionávelAcesso rápido por índice — get(i)
LinkedListNós encadeados (referências)Inserir/remover nas pontas
 
List<String> lista = new ArrayList<>();
lista.add("Ana");        // adiciona no fim
lista.add("Bia");
lista.get(0);            // lê por índice → "Ana"
lista.remove(0);         // remove por índice
lista.size();            // quantidade de elementos
💡 Regra prática: muito get(i) → ArrayList. Muita inserção/remoção no começo/fim → LinkedList.
📚 Pilha (Stack) LIFO
📌 LIFO — Last In, First Out: o último que entra é o primeiro que sai (como uma pilha de pratos).
MétodoO que faz
push(x)Insere no topo
pop()Remove e retorna o topo
peek()Vê o topo sem remover
isEmpty()Verifica se está vazia
 
Stack<Integer> pilha = new Stack<>();
pilha.push(1); pilha.push(2); pilha.push(3);
System.out.println(pilha.pop());  // 3 (o último que entrou)
📎 Da sua nota: Stack está em java.util e herda de Vector. pop() em pilha vazia lança EmptyStackException. push é O(1). Aplicação clássica: Call Stack (controle de chamadas de métodos) e avaliação de expressões.
🎫 Fila (Queue) FIFO
📌 FIFO — First In, First Out: o primeiro que entra é o primeiro que sai (como fila de banco).
MétodoO que faz
offer(x)Adiciona no fim
poll()Remove e retorna o primeiro (null se vazia)
peek()Vê o primeiro sem remover
 
Queue<Integer> fila = new LinkedList<>();
fila.offer(1); fila.offer(2); fila.offer(3);
System.out.println(fila.poll());  // 1 (o primeiro que entrou)
📎 Da sua nota: implementações comuns são ArrayDeque, LinkedList e PriorityQueue (esta entrega sempre o de maior prioridade, não a ordem de chegada).
